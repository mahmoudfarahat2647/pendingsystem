# Playwright E2E Conventions

Project-specific rules for all E2E tests in this repository.

---

## 1. Always Use the `authedPage` Fixture

Import `test` and `expect` from `e2e/fixtures/index.ts`, never directly from `@playwright/test`.
The `authedPage` fixture loads the admin session from `e2e/.auth/admin.json` automatically.

```ts
import { test, expect } from "../fixtures";

test("example", async ({ authedPage }) => {
  await authedPage.goto("/orders");
});
```

---

## 2. Selector Priority

1. `data-testid` â€” preferred
2. ARIA role via `getByRole()`
3. Element `id`
4. Never CSS classes or implementation-detail selectors

```ts
// Good
page.getByTestId("submit-button");
page.getByRole("button", { name: "Save" });

// Bad
page.locator(".btn-primary");
page.locator("div > span.label");
```

---

## 3. No `waitForTimeout`

Never use `page.waitForTimeout()`. Use intent-based waits instead:

```ts
// Good
await page.waitForURL("/dashboard");
await page.waitForSelector("[data-testid='order-table']");
await expect(locator).toBeVisible();

// Bad
await page.waitForTimeout(2000);
```

---

## 4. One `describe` Per Feature, One `test` Per Scenario

```ts
test.describe("Order creation", () => {
  test("creates a new order with valid data", async ({ authedPage }) => { ... });
  test("shows validation error when part number is missing", async ({ authedPage }) => { ... });
});
```

---

## 5. File Naming

| Type        | Pattern                        |
|-------------|--------------------------------|
| Smoke       | `feature.smoke.spec.ts`        |
| Regression  | `feature.regression.spec.ts`   |

Examples: `orders.smoke.spec.ts`, `booking.regression.spec.ts`

---

## 6. Seed Data in `e2e/seeds/`

Each seed scenario lives in its own file under `e2e/seeds/`.
Use the `dbSeed` fixture (opt-in) to run setup before a test.

```ts
// e2e/seeds/orders-with-reminders.ts
export async function seedOrdersWithReminders() { ... }
```

---

## 7. Auth State from `e2e/.auth/admin.json`

Auth state is generated by `e2e/global-setup.ts` and stored at `e2e/.auth/admin.json`.
Never hardcode credentials in test files. Always rely on the `authedPage` fixture.

---

## 8. Screenshots on Failure Only

Do not add `page.screenshot()` calls in final specs.
Configure failure screenshots globally in `playwright.config.ts`:

```ts
use: {
  screenshot: "only-on-failure",
}
```

Remove any debug screenshots (`debug-*.png`) before committing.

---

## 9. Tests Must Be Fully Independent

- Each test sets up its own state â€” no shared mutable state between tests
- No test depends on the execution order of another test
- Use `dbSeed` or per-test `beforeEach` hooks to establish preconditions

---

## 10. Prefer `getByRole` and `getByTestId`

Use Playwright's semantic query methods over raw `locator()`:

```ts
// Preferred
page.getByRole("dialog", { name: "Confirm Delete" });
page.getByRole("row", { name: "Part #1234" });
page.getByTestId("order-actions-menu");

// Avoid
page.locator("[role='dialog']");
page.locator("#order-actions-menu");
```
